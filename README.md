# SAP-1 in Verilog

An 8-bit SAP-1-style computer implemented in Verilog. The design combines a small shared-bus CPU, 16 bytes of programmable RAM, a six-step controller, and a hexadecimal seven-segment display interface. In addition to the traditional load, add, subtract, output, and halt operations, this version supports signed and unsigned multiplication and division.

## Features

- 8-bit data path and shared internal bus
- 4-bit program counter and 16 x 8-bit RAM
- Six timing states for instruction fetch and execution
- Manual RAM programming through switches and buttons
- Accumulator, B register, instruction register, memory address register, and output register
- Signed `MUL` and `DIV` using 8-bit two's-complement values
- Unsigned `UMUL` and `UDIV`
- LED bus/output monitoring and multiplexed hexadecimal seven-segment output
- Self-checking testbenches for load, output, arithmetic, overflow, and division-by-zero behavior

## Architecture

[`SAP1.v`](SAP1.v) is the top-level module. It connects the processor data path, control unit, RAM, clock divider, and display logic.

The controller uses the six states generated by [`ring_counter.v`](ring_counter.v). States T1-T3 fetch an instruction; states T4-T6 execute it. The upper nibble of each instruction is decoded as the opcode, while memory instructions use the lower nibble as a RAM address.

| Opcode | Mnemonic | Operation |
|---|---|---|
| `0x0` | `OUT` | Copy the accumulator to the output register |
| `0x4` | `HLT` | Stop the processor clock |
| `0x5` | `ADD addr` | `A <- A + RAM[addr]` |
| `0x7` | `SUB addr` | `A <- A - RAM[addr]` |
| `0x8` | `MUL addr` | Signed `A <- A * RAM[addr]` (low 8 bits) |
| `0x9` | `DIV addr` | Signed `A <- A / RAM[addr]` |
| `0xA` | `UMUL addr` | Unsigned `A <- A * RAM[addr]` (low 8 bits) |
| `0xB` | `LDA addr` | `A <- RAM[addr]` |
| `0xC` | `UDIV addr` | Unsigned `A <- A / RAM[addr]` |

For multiplication, the internal `Cout` signal indicates that the full product does not fit in 8 bits. Division by zero produces `8'hFF` and sets `Cout`.

## Top-Level Interface

The `SAP1` module exposes a switch/button interface suitable for an FPGA board with 16 switches, five buttons, 16 LEDs, and a four-digit seven-segment display.

| Input | Function |
|---|---|
| `SW[15:12]` | RAM address while programming |
| `SW[11]` | Mode: `0` programs RAM, `1` runs the CPU |
| `SW[10]` | Show the selected RAM byte while in programming mode |
| `SW[7:0]` | RAM data byte while programming |
| `BTN[1]` | Write the selected byte to RAM |
| `BTN[2]` | Clear RAM while in programming mode |
| `BTN[4]` | Reset the processor registers and program counter |
| `CLK` | External clock input |

While the CPU is running, `LED[15:8]` shows the internal bus and `LED[7:0]` shows the output register. The seven-segment display shows the output register as two hexadecimal digits. In programming mode it can show the selected RAM address and byte.

The default `CLOCK_DIVIDER_BIT` parameter is `22`. Testbenches override it with `0` to accelerate simulation.

## Example Program

This program computes `3 + 4`, sends the result to the output register, and halts:

| RAM address | Byte | Meaning |
|---|---|---|
| `0x0` | `0xB4` | `LDA 4` |
| `0x1` | `0x55` | `ADD 5` |
| `0x2` | `0x00` | `OUT` |
| `0x3` | `0x40` | `HLT` |
| `0x4` | `0x03` | First operand |
| `0x5` | `0x04` | Second operand |

To enter it through the top-level interface, keep `SW[11]` low, select each address with `SW[15:12]`, place the byte on `SW[7:0]`, and pulse `BTN[1]`. Set `SW[11]` high to run the program.

## Simulation

### Prerequisites

Use a Verilog simulator such as [Icarus Verilog](https://steveicarus.github.io/iverilog/). No simulator-specific project files are required.

Compile a testbench from the repository root by selecting its top-level module. For example, to run the comprehensive arithmetic testbench:

```sh
iverilog -g2012 -s SAP1_tb -o SAP1_tb.vvp *.v
vvp -n SAP1_tb.vvp
```

Replace `SAP1_tb` with another testbench module to run a focused test:

| Testbench | Coverage |
|---|---|
| `Load_tb` | `LDA` instruction |
| `Out_tb` | `OUT` instruction and LED output |
| `AddSub_tb` | `ADD` and `SUB` instructions |
| `SAP1_tb` | Signed/unsigned multiply and divide, overflow, and division by zero |
| `SAP1_presentation_tb` | Small `3 + 4` demonstration program |

The testbenches print `TEST PASSED`, `TEST FAILED`, `DEMO PASSED`, or `DEMO FAILED` messages to the simulator output.

## FPGA Integration

Add all non-testbench `.v` files to your FPGA project and select `SAP1` as the top module. The repository does not include a board constraint file or vendor project, so clock and I/O pins must be assigned for the target board. Adjust `CLOCK_DIVIDER_BIT` if a different visible CPU clock rate is needed.
